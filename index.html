01 / Capabilities

four layers between the camera
and the attendance log.

Every recognized face passes through the full pipeline before it lands on disk. Detection finds the face. Anti-spoofing decides whether it's real. Recognition identifies who it is. Notification tells the people who care.

[ 01 ]

face detection

HAAR CASCADE / 30 FPS

Frames pull from the webcam at 30fps. A Haar Cascade classifier locates faces in greyscale, returning bounding boxes for every face in the frame, with padding and minimum-size filters tuned for typical doorway distance.

[ 02 ]

anti-spoofing

KERAS CNN / 160×160

Every detected face is resized to 160×160 and run through a trained binary classifier that distinguishes a living face from a printed photo or screen replay. Scores above the threshold are dropped before they ever reach the recognition stage.

[ 03 ]

attendance state

LOGIN ⇄ LOGOUT / 5S COOLDOWN

Each recognized identity toggles between login and logout in a plain-text state file. A five-second cooldown per person prevents the same face from being marked twice while they stand at the door. Append-only history goes to attendance.txt with timezone-aware timestamps.

[ 04 ]

guardian notify

SMTP / TLS / OPTIONAL

When enabled, every successful mark fires an SMTP email over TLS to the address registered against that identity — name, roll number, action, date, time. Toggle on or off per deployment with a single config flag.

02 / Pipeline

from photon to permanent record.

A single frame flows through six stages. Each stage hands off only the data the next one needs. Latency budget: well under 100ms per recognized identity on a mid-spec laptop.

04 / Deploy

three commands. one camera.

Local edition runs on any machine with Python 3.10+ and a webcam. No cloud setup, no API keys (unless you turn on email).

01

clone the repo

Grab the source and step into the directory. The Haar Cascade and anti-spoofing model are included.

02

install dependencies

OpenCV, face_recognition (dlib), TensorFlow, pytz. Use a virtualenv. dlib will compile on first install — give it a few minutes.

03

run

Drop reference photos of each person into face_recognition/ named after them. Run main.py. Press Q to quit, S to capture, R to reload faces.

terminal // QUICKSTART 
# 01 — clone
git clone https://github.com/TenzyBravo/face-attendance-system.git
cd face-attendance-system

# 02 — install (virtualenv recommended)
python -m venv venv
source venv/bin/activate   # windows: venv\Scripts\activate
pip install -r requirements.txt

# 03 — run
python main.py

# controls inside the window:
#   q  → quit
#   s  → manual capture
#   r  → reload known faces
05 / Deployment Zones

where this earns its keep.

Built for environments where attendance matters enough to automate, but not enough to outsource. Schools, small offices, restricted-access rooms, anywhere the alternative is a paper sheet by the door.

schools

Replace paper registers. Notify parents the moment a student arrives or leaves.

offices

Frictionless clock-in for small teams. No fobs, no apps, no fingerprint readers.

restricted

Server rooms, vaults, archives. Anti-spoofing rejects photo and screen-replay attempts.

events

Pre-registered guest entry. The reference photo is the ticket.
